Sistema de Reconhecimento de Sentimentos

(Especificação de Requisitos)

A - VISÃO GERAL DO SISTEMA

O Sistema de Reconhecimento de Sentimentos em consistem uma web application no qual um usuário pode se cadastrar na plataforma e fazer análises de frases, gerar frases baseadas em sentimentos escolhidos, ver as frases que já foram análisadas e até mesmo ajudar o algoritmo de reconhecimento das frases. O sistema ainda conta com informações entre outros aspectos relacionados aos sentimentos contidos nas frases.

B - REQUISITOS FUNCIONAIS
B1 - Cadastros

  1. O sistema deve permitir o cadastro de usuários da plataforma. Este usuários devem ter os seguintes atributos: nome, email, data de nascimento, celular,usuário, senha. Os usuários são identificados pelo usuário e podem utilizar o email ou usuário para fazer login na plataforma. Os usuários podem também alterar nome, email, data de nascimento, celular e senha. Eles também podem excluir a conta, realizando a exclusão dos dados armazenados. Para realizar estes procedimentos, o usuário deve estar logado na plataforma.

  2. O sistema deve também permitir que administradores cadastrados no sistema possam banir/excluir qualquer usuário cadastrado no sistema. Para isto o admnistrador deve estar logada na plataforma.

  3. O sistema deve permitir que um usuário logado na plataforma possa digitar uma frase e submetê-la a um algoritmo de reconhecimento de sentimentos já préviamente treinado. Este algoritmo então deverá classificar a frase e a aplicação web deve mostrar ao usuário qual o sentimento correspondente aquela frase. Com isto o usuário deve classificar se o sentimento identificado corresponde ou não ao real sentimento da frase. Frases classificadas corretamente servirão para melhorias no algoritmo de reconhecimento dos sentimentos.

  4. O sistema deve permitir que um usuário logado possa recuperar uma frase aleatória do sistema de acordo com um sentimento que ele escolher. Esta escolha se dá através de emoticons representando os sentimentos.

  5. O sistema deve permitir que um usuário logado possa visualizar o histórico de frases que ele analisou bem como os resultados delas. Este histórico deve ser mostrado em ordem cronológica. O usuário pode também limpar este histórico por completo ou deletar frases individualmente.

  6. O sistema deve permitir que o usuário possa ajudar o algoritmo de reconhecimento por meio de inserção de uma frase e selecionando entre os sentimentos definidos qual melhor se encaixa para ela.

  7. Análises submetidas e não realizadas por completo, como selecionar se o sentimento reconhecido é correto ou não, pode ser vista nos histórico.

  8. O sistema deve permitir que um usuário administrador possa cadastrar novos sentimentos. Os sentimentos devem ter nome, emoticon em imagem, cor relacionada, descrição.

B2 - Restrições do modelo.

  1. O sistema deve retreinar o modelo quando o número de frases classificadas corretamente (requisito B1.3) somado ao número de frases fornecidas pelos usuários (requisito B1.6) chegar a 25.

  2. O sistema deve excluir as frases digitadas e análisadas ou frases dadas como entrada para ajudar o classificador caso o usuário seja banido/excluído ou tenha realizado a requisição de deletar a conta. Quando isto ocorrer um novo modelo deverá ser gerado sem estes dados, contudo, até que o modelo não esteja pronto o antigo deverá ser utilizado.

B3 - Impressão de Análises

  1. O sistema deve permitir ao usuários exportar as análises feitas por ele em formato csv, excel ou json.

C - REQUISITOS NÃO FUNCIONAIS
C1 - Confiabilidade

  1. O sistema deve ter capacidade para reconhecer o sentimento e armazenar a análise, caso a submissão da frase tenha sido feita corretamente, mesmo que o usuário não permaneça na tela de carregamento, neste caso a frase é classificada como incorreta para não correr o risco de usá-la como correta no retreinamento, desta forma a frase fica disponível no histórico de análises.

  2. O sistema deve possuir senhas de acesso e identificação para diferentes tipos de usuários: administrador do sistema e usuários em geral.

C2 - Eficiência

  1. O sistema deve trazer ao usuário a análise on-line em menos de 10 segundos.

  2. O sistema deve trazer ao usuário as análises realizadas por ele em menos de 10 segundos.

  3. O treinamento do modelo pode demorar o tempo que for necessário, enquanto não for gerado o antigo modelo deve ser usado.

C3. Portabilidade

  1. Tratando-se de uma aplicação web, o sistema deve ser executado em qualquer dispositivo.

  2. O back-end deve rodar em um servidor utilizando Linux Ubuntu 20.04 LTS.

C4. Técnologias envolvidas

  1. O back-end da aplicação deve utilizar Java 8 e o banco de dados ser em MySQL. A comunicação entre o front e os dados é feita através do GraphQL.

  2. O front-end é modelado utilizando o Bootstrap, CSS e HTML.

  3. Os modelos de treinamento serão feitos utilizando o Weka 3.8.4 e Java 8.

(Diagramas de Caso de Uso)

Cadastro de Novo Sentimento

Descrição: Diagrama correspondente ao caso de uso de cadastrar um novo sentimento. Este procedimento se dá através do Administrador que cadastra os dados referentes ao um novo sentimento, este sentimento então é buscado para verifica a existencia ou não dele, caso seja um novo sentimento ele é autorizado a cadastrá-lo. Também são feitas verificações de imagem, certificando se está na proporção e tamanho correto. Também são verificadas as cores correspondente ao sentimento, devendo serem disponibilizadas pelo código hexadecimal.

CÓDIGO FONTE: 

@startuml

actor Administrador

Administrador --> (CadastrarSentimento) : << dados do sentimento >>
(CadastrarSentimento) --> (ConsultaSentimentosExistentes) : << include >>
(ConsultaSentimentosExistentes) --> (ValidaNovoSentimento) :<< include >>
(CadastrarSentimento) --> (ValidaImagem) : << include >>
(CadastrarSentimento) --> (ValidaCor) : << include >>

@enduml

Análise do Sentimento de Frase

Descrição: Diagrama correspondente a sequência de funcionamento do processo que um usuário realiza para analisar um frase que ele deseja saber. O processo funciona com o usuário abrindo a página inicial e selecionando a opção de analisar, então uma segunda página com uma caixa de texto será fornecida e após ele digitar e submeter o texto, será carregado uma terceira página onde estará a análise e uma opção de classificar se o sentimento devolvido está correto ou não.

CÓDIGO FONTE: 

@startuml
actor Usuário

collections PaginaInicial
collections PaginaReqAnalise
collections PaginaAnaliseConcluida
collections EnviaAnalise
collections PréProcessaTexto
collections GeraClasseAnalise
collections BancoDeDados

PaginaInicial --> PaginaInicial : aguarda_opcao()
PaginaInicial --> PaginaReqAnalise : abre_pagina()
PaginaReqAnalise --> PaginaReqAnalise : aguarda_texto()
Usuário --> PaginaReqAnalise : digita_texto()
Usuário --> PaginaAnaliseConcluida : aguarda_analise()
PaginaReqAnalise --> EnviaAnalise : envia_texto()
EnviaAnalise --> PréProcessaTexto : processa_texto()
PaginaReqAnalise --> BancoDeDados : armazena_texto()
PréProcessaTexto --> GeraClasseAnalise : classifica_texto()
GeraClasseAnalise --> PaginaAnaliseConcluida : retorna_classe()
PaginaAnaliseConcluida --> PaginaAnaliseConcluida: aguarda_certificacao_analise()
Usuário --> PaginaAnaliseConcluida : classifica_analise_feita()
PaginaAnaliseConcluida --> BancoDeDados : armazena_analise_concluida()

@enduml

Cadastro Usuário

Descrição: Diagrama correspondente a sequência de funcionamento do processo de cadastro de usuário. O processo funciona com o usuário abrindo o webapp e selecionando onde será levado a tela de cadastro, caso não tenha sido cadastrado, então uma função analisa se o login e já existe e se não, continua o cadastro e se já, volta pra tela inicial .

Histórico de Análise

Descrição: Diagrama correspondente a sequência de funcionamento do processo do histórico de análise. O processo funciona com o usuário abrindo o webapp e selecionando onde será levado a tela de histórico, e esta pesquisará no banco todas as frases cadastradas e, ao selecionar uma dessas frases trazidas, será feita nova pesquisa no banco que retornará o sentimento correspondente à aquela frase .
