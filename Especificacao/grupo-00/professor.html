Sistema para Locadora de Carros

(Especificação de Requisitos)

A - VISÃO GERAL DO SISTEMA

O sistema para a Locadora de Carros Ubirajara consiste do gerenciamento de locação de carros, bem como do controle de itens que podem ser adicionados ao aluguel, tais como combustível, seguros (danos materiais e pessoais), taxa de retorno, cadeiras de bebê, motoristas, entrega do carro, etc. O sistema deve ainda emitir diversos tipos de relatórios e consultas, possibilitando um melhor gerenciamento das locações.

B - REQUISITOS FUNCIONAIS
B1 - Lançamentos diversos

  1. O sistema deve permitir a inclusão, alteração e remoção de clientes da locadora de carros com os seguintes atributos: nome, endereço (rua, número, bairro, cep, cidade, estado), contato (telefone residencial, telefone comercial, celular, e-mail), cpf (para pessoa física), data de nascimento e cnpj (para pessoa jurídica). Um cliente pode ter mais de um endereço para correspondência. Um cliente tem apenas um contato.

  2. O sistema deve permitir a inclusão, alteração e remoção de clientes da locadora de carros com os seguintes atributos: nome, endereço (rua, número, bairro, cep, cidade, estado), contato (telefone residencial, telefone comercial, celular, e-mail), cpf (para pessoa física), data de nascimento e cnpj (para pessoa jurídica). Um cliente pode ter mais de um endereço para correspondência. Um cliente tem apenas um contato.

  3. O sistema deve permitir a inclusão, alteração e remoção das diversas categorias de carros, com os seguintes atributos: identificador da categoria, descrição da categoria e preço diário de locação. Para cada categoria podem existir diversos carros.

  4. O sistema deve permitir a inclusão, alteração e remoção dos carros para aluguel pertencentes à locadora de carros. Cada carro possui os seguintes atributos: nome, placa, marca, ano, quilometragem atual, uma identificação de patrimônio (ativo ou não), um status de locação (alugado ou disponível), o número máximo de dias que este carro pode ficar com o cliente e código para uma etiqueta QR Code.

  5. O sistema deve permitir a inclusão, alteração e remoção dos tipos de serviços adicionais oferecidos pela locadora de carros. Cada tipo de serviço adicional possui os seguintes atributos: identificador do serviço oferecido, descrição do serviço e preço.

  6. O sistema deve permitir a locação de carros para seus clientes. O cliente pode selecionar um carro por meio das seguintes opções: (i) atendente via balcão; (ii) terminal ATM disponível no ambiente interno da locadora; e (iii) aplicativo móvel (mobile app). Em ambos os meios de consulta, o cliente pode visualizar todas as informações relativas ao carro que selecionou. Além disso, vale destacar que uma locação pode conter mais do que um carro. Em contrapartida, uma locação deve ser registrada nominalmente a um cliente. Dessa forma, cada carro pode ter uma data de devolução específica. A locação só deve ser permitida se houverem carros do tipo requisitado disponíveis no período indicado. O sistema não permite agendamento de carros em períodos futuros.

  7. O sistema deve permitir a devolução de um ou mais carros por um cliente. Para isso, devem ser registrados a data e hora de devolução do carro, quilometragem registrada no momento da devolução e número de diárias cobradas. No final, o sistema deve também apresentar na tela o total a pagar. Uma locação é finalizada quando todos os carros contidos naquela locação forem devolvidos. Para isso, o sistema deve indicar o status da locação.

  8. O sistema deve permitir as seguintes opções de pagamento do aluguel do carro: 1) à vista (em dinheiro ou cartão de crédito); 2) parcelado em até 5 vezes. O sistema deve gerenciar as parcelas, estabelecendo datas de vencimento e de pagamento. Parcelas pagas em atraso devem ser pagas com juros de 0,5% ao dia com multa de 5% ao mês.

  9. O sistema deve permitir a quitação de uma fatura paga pelo cliente, contendo as seguintes informações: identificador da fatura, data de vencimento, data de pagamento, valor total pago.

B2 - Impressão de diversos tipos de relatórios e consultas

  1. O sistema deve permitir a impressão de uma listagem dos carros alugados no momento, contendo o nome do cliente, placa, marca e modelo do carro, data de retirada e data prevista para devolução de cada carro.

  2. O sistema deve permitir a impressão de uma listagem das locações efetuadas para a data atual, contendo o nome do cliente, telefone para contato e carro(s) (nome e placa).

  3. O sistema deve permitir a impressão de um relatório contendo as faturas em atraso no período (por exemplo, semanal ou quinzenal), contendo, para cada dia do período, o nome do cliente, a data de vencimento e o valor devido pelo cliente.

C - REQUISITOS NÃO FUNCIONAIS
C1 - Confiabilidade

  1. O sistema deve ter capacidade para recuperar os dados perdidos da última operação que realizou em caso de falha.

  2. O sistema deve fornecer facilidades para a realização de backups dos arquivos do sistema.

  3. O sistema deve possuir senhas de acesso e identificação para diferentes tipos de usuários: administrador do sistema, funcionários da locadora de carros e clientes que têm acesso ao sistema na locadora de carros (em quiosques especiais).

C2 - Eficiência

  1. O sistema deve responder a consultas on-line em menos de 5 segundos.

  2. O sistema deve iniciar a impressão de relatórios solicitados dentro de no máximo 20 segundos após sua requisição.

C3. Portabilidade

  1. O sistema deve ser executado em computadores Pentium 200mHz ou superior, com sistema operacional Windows 98 ou acima.

  2. O sistema deve ser capaz de armazenar os dados em base de dados Oracle ou Sybase.
